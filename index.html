Professional Experience and Skills



I am a core reviewer in the OpenStack community for StoryBoard, a task-tracker designed for cross-project work. It's over here!

I currently work for Codethink Ltd. I have experience with:

OpenStack

I am active in the community, and am familiar with the processes, terminology, and many of the tools.

As a core reviewer on the StoryBoard project, part of the OpenStack project's infrastructure, I have gained a working knowledge of the the infrastructure configuration and documentation, along with cross-project initiatives. I have a high-level view of how OpenStack core services fit together, along with the roles of many other projects, and I have an understanding of OpenStack governance.

In October 2015, I co-moderated a task-tracking infrastructure session in Tokyo with Thierry Carrez (ttx) at the Mitaka summit.
I also co-led the Newton task-tracking infrastructure session in April 2016 with Adam Coldrick (SotK) and Craige McWhirter (craige).

I've somehow ended up responsible for announcing all StoryBoard news; as a result, I have been featured in the OpenStack SuperUser magazine on two occasions. My stackalytics are here. I review far more than I code; StoryBoard is continuously delivered, so I am meticulous about testing and reviewing patches, to ensure that the infrastructure production instance always functions as expected.

Linux

Common tools, the shell, the Filesystem Hierarchy Standard. I do the vast majority of my work on Linux.

Git

I use Git almost every day of my professional life. Some commands I use a lot: `clone`, `branch`, `checkout`, `add`, `rm`, `commit`, `push`, `pull`, `merge`, `log`, `status` and `rebase` (and variations on a theme of `git reset --hard HEAD`, sadly). I'm also familiar with common flags. I use Git on both Linux and Windows.

Gerrit

I LOVE GERRIT WITH ALL MY HEART. I use it every day for code review and have written scripts that use the API. (One reason I love Gerrit so much is that the API makes it simple to do this) I've also helped test the OpenStack infrastructure staging instance (I uncovered a bug in something completely unrelated to what we were testing, which is how these things always seem to go...).

Python

For professional work, I mainly use python for scripting; there are example tutorial scripts I wrote for others here. Those are commented versions of the scripts I made to help with the infrastructure team's September bugsquash.

I like python, as it pushes people to write code that is easy to understand; communication matters to me.
I'm learning python in more depth at the moment, so that I can feel more comfortable with the libraries, and classes and object-oriented design more generally; you can follow my progress on Github.
